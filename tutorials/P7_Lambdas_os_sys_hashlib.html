Základné moduly Pythonu 2- os, sys, hashlib a lambda funkcie

  • Modul sys

    • Prvý modul na ktorý sa pozrieme je modul sys, ktorý má za úlohu Sprostredkovať našemu programu dôležité údaje o systéme. Modul si naimportujeme príkazom:

    import sys

    Ak chceme vedieť na akom druhu operačného systému program beží získame tento údaj pomocou premennej: br>
    sys.platform

    Rozlišnosti v syntaxe jednotlivých verzií Pythonu môžu spôsobiť, že kód napísaný pre Python 3.8 nebude korektne pracovať na verzií Python 3.6 a preto je potrebné získať informáciu o Python Interpreteri na ktorom program beží, ktorá je uložená v premennej:

    sys.version_info

    Modul tiež umožňuje získavať informácie o aktuálne nainštalovaných moduloch jazyka Python a o cestách k nim. Napríklad pomocou premenných:

    sys.modules.keys()
    sys.path

    Často využívanou premennou je:

    sys.argv

    Informácia pokiaľ ju spustíte z interkatívnej konzoly bue obsahovať [] čo znamená prázdny zoznam - LIST. Ale pokiaľ skript spustíte pomocou príkazu z príkazového riadku Linuxu alebo Powershellu:

    python3 program.py ARGUMENT1 ARGUMENT1

    poskytne Vám zoznam, ktorý bude obsahovať ako prvý argument: sys.argv[0]názov spusteného programu
    sys.argv[1]ARGUMENT1
    sys.argv[2]ARGUMENT2
    Toto nám umožní spúšťať pythonovský program aj so zadaním vstupných PARAMETROV a ARGUMENTOV priamo z príkazového riadku !!! Rovnako ako pri Linuxových a Powershellových príkazoch.

  • Modul os

    • Modul slúži na manipuláciu Operačného systému a poskytuje mocné nástroje pre prácu s terminálom, súbormi, adresármi, PERMISSIONS atď. Naimportujeme si ho:

    import os

    Hneď ako prvý si vyskúšame najmocnejšiu z jeho metód, pomocou ktorej vieme Linuxu alebo Powershellu zadať akýkoľvvek príkaz OS na vykonanie:

    os.sys("ls -al")

    Napríklad vypísanie podrobností adresára v ktorom sa nachádzame. Príkaz sa vykoná a program pokračuje ďalej.

    P O Z O R !!!

    Príkaz nepovie Pythonu či sa vykonal korektne a je preto potrebné to skontrolovať nejakou inou funkciou.

    Touto metódou by napríklad mohla byť metóda:

    os.listdir(path="/home/")

    Táto vráti obsah adresára podľa príslušnej cesty PATH v podobe zoznamu LIST. Môžeme ho pomocou príkazu Pythonu spracovať, prehľadať, porovnať atď. Môžeme si zobraziť aktuálny pracovný adresár CURRENT WORKING DIRECTORY pomocou metódy:

    os.getcwd()

    Modul os však ide omnoho ďalej, disponuje množstvom metód, ktoré umožňujú Pythonu kontrolovane presúvať súbory, meniť nastavenia a pod. Užitočným je tiež prehliadanie stromovej štruktúry adresárov a súborov pomocou metódy:

    os.walk(top="PATH")

    Ak si ho dáme zobraziť pomocou príkazu print zobrazí nám informáciu o existujúcom GENERATOR objekte a jeho umiestnení v pamäti počítača.

    GENERÁTORY

    Python podporuje vytváranie generátorov, ktoré majú oproti ukladaniu do premenných množstvo výhod. As najväčšou je to, že nezaberejú toľko miesta v pamäti a sú AdHoc generované pre potreby programu. GENERÁTORY vytvoríme jednoducho ak miesto return na konci funkcie alebo metódy použijeme yield. To znamená, že Python vygeneruje hodnotu a poskytne ju programu v podobe GENERATOR objektu v pamäti. POZOR ak chcete jeho obsah zobraziť musíte ho zobraziť ako list(GENERATOR).

    Tento krátky kód Pythonu nám pomôže zobraziť obsah:

    content = os.walk(top="/home/andrejli/Projects/PythonScratch")
    for dirs,files,roots in content:
       print(dirs,files,roots)

    S výstupom možno pracovať podľa potrieb ho prispôsobovať a všemožne ho použiť.

  • Modul hashlib

    • Posledným modulom, ktorý si v tejto lekcií predstavíme je modul hashlib, ktorý budeme potrebovať pri vypočítavaní HASHSTRINGOV keď budeme programovať náš vlastný lúskač kódov v ďalšej časti kurzu.

    ČO JE TO HASHSTRING

    HASHSTRING je výsledok kryptografického algorytmu, ktorý vypočítava jedinečný reťazec pre akékoľvek dáta. Výsledok je jedinečný a unikátny a preto prezentuje niečo ako digitálny odtlačok prsta dát.
    Tieto algorytmy majú široké uplatnenia pri ukladaní hesiel, kontroly zmeny dát a ich anonymizácií.

    P O Z O R ! ! !

    Hashovací algorytmus SHA1 bol prelomený a podarilo sa spätne vypočítať dáta, ktoré boli použité na jeho výpočet. Používajte iba spoľahlivé neprelomené hashovacie algorytmy!!!

    Modul si naimportujeme do Pythonu príkazom:

    import hashlibr

    Ako prvé musíme vytvoriť inštanciu hashovacieho algorytmu:

    h = haslib.md5()

    Pre tento prípad použijeme algorytmus MD5. Algorytmus požaduje ako vstupné dáta nie v premennej STRING ale BYTES, ktoré sú zjednodušene hexadecimálnymi alebo binárnymi hodnotami dát alebo STRINGU. Získame ich pomocou zmenou STRINGU na BYTES s doplnením KÓDOVANIA znakov. Pre tento prípad použijeme kódovanie znakov utf8:

    pripraveny_text = bytes("HELLO WORLD !!!", encoding="utf8")

    Pripravený text vložíme do algorytmu, ktorý doposiaľ nemal čo počítať. Použijeme pritom objekt h, ktorý je inštanciou triedy hashlib.md5. Pomocou metódy update pridáme našu premennú pripraveny_text:

    h.update(pripraveny_text)

    Nakoniec si necháme zobraziť vypočítaný HASHSTRING v hexadecimálnej podobe:

    print(h.hexdigest())

    Tento reťazec je unkátnou prezentáciou reťazca HELLO WORLD !!! a kdekoľvek a kedykoľvek ho opäť vypočítame bude ZHODNÝ.

  • Funkcie LAMBDA

    • V lekcií 4 sme sa naučili vytvárať vlastné funkcie pomocou príkazu def. Python v prípade jednoduchších funkcií umožňuje použiť tzv. lambda funkcie. Zadefinujeme si funkciu, ktorú si aj ponmenujeme:

    sucet = lambda x,y: x + y

    V tomto jedinom riadku sme zadefinovali premenné x a y ako aj samotnú funkciu x + y. Ak ju chceme použiť, môžeme ju jednoducho napísať:

    sucet(10,15)

    ALEBO

    sucet("Hello"," World !!!")

    math.pi
